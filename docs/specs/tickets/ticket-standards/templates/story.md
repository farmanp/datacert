# AI-Ready Story Template (Core)

## 1. Intent (Required)
**User Story:**
As a [user or system]
I want [capability]
So that [value]

**Success Looks Like:**
[One sentence describing the desired end state]

## 2. Context & Constraints (Required)
**Background:**
[Why this is needed]

**Scope:**
- **In Scope:** [What we are doing]
- **Out of Scope:** [What we are NOT doing]

**Constraints:**
- [Tech / Legal / UX / Performance]

## 3. Acceptance Criteria (Required)
*Format: Gherkin (Given/When/Then)*

**Scenario: [Behavior Name]**
Given [initial state]
And [preconditions]
When [action or event]
Then [expected outcome]
And [side effects]

## 4. AI Execution Instructions (Required)
**Allowed to Change:**
- [UI / API / DB / Config]

**Must NOT Change:**
- [Public Contracts / Data Models / Existing Behavior]

**Ambiguity Rule:**
If unclear, acceptance criteria override all other sections.

## 5. Planned Git Commit Message(s) (Required)
*Format: Conventional Commits (<type>(<scope>): <subject>)*
*Example: feat(auth): allow users to reset password via email*

- [commit 1]

## 6. Verification & Definition of Done (Required)
- [ ] Acceptance criteria pass
- [ ] Tests written and passing
- [ ] Code reviewed
- [ ] No breaking changes
